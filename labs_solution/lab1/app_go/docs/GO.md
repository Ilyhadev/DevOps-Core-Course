# Go Justification

Why Go for the bonus task:

- Small, efficient static binaries that simplify deployment and multi-stage Docker builds.
- Fast compilation and a simple standard library that includes a high-quality `net/http` server.
- Strong ecosystem for building lightweight services and command-line tools.

Comparison to Python (short):

- Binary size: Go produces a single binary (larger than a tiny script but simpler to distribute).
- Performance: Go's concurrency model and compiled nature offer better throughput for CPU-bound or concurrent workloads.
- Development speed: Python requires less code for quick scripts, but Go provides safer static types and easier cross-compilation.
